# Work Center Desk — Landing Page

Landing page desenvolvida como parte de um desafio técnico de Frontend, com foco em UI/UX, responsividade e organização de código.

---

## Objetivo do Projeto

Construir uma landing page para um produto SaaS fictício (Work Center Desk), voltado para gestão de tarefas de times remotos, com foco em:

- Experiência do usuário (UX)
- Interface moderna e consistente (UI)
- Responsividade (desktop e mobile)
- Código limpo e organizado

---

## Sobre o Produto

O Work Center Desk é uma plataforma de produtividade para equipes de tecnologia, semelhante a ferramentas como Notion, Trello e Asana.

O público-alvo são gestores de times, que precisam de uma interface clara, rápida e eficiente para tomada de decisão.

---

## Decisões de UI/UX

Pensando no público do produto (gestores de times de tecnologia), tomei a decisão de priorizar uma interface limpa e objetiva. Apesar disso, tomei cuidado para que não ficasse monótona e desinteressante, estruturando os elementos de tal maneira que chamassem a atenção do usuário. Desse modo, organizei a landing page com uma hierarquia visual bem definida, utilizando títulos claros, pouco texto e seções organizadas, permitindo que o usuário compreenda rapidamente o valor da ferramenta sem precisar ler muito. Além disso, destaquei os botões com as principais ações, com o intuito de facilitar a tomada de decisão do usuário.

---

## Orgulho Técnico

A parte do código da qual mais me orgulho foi a implementação da responsividade da landing page utilizando apenas HTML e CSS. Utilizei Flexbox para estruturar os layouts e media queries para adaptar os componentes para diferentes tamanhos de tela, especialmente no mobile. Também destaco a construção de componentes reutilizáveis, como botões e cards, com uso de variáveis CSS para cores e tipografia, o que facilita manutenção e consistência visual em toda a aplicação. Outro ponto importante foi a implementação da navegação responsiva com menu mobile (hamburger), garantindo boa usabilidade em telas menores.

---

## Autocrítica

Com mais tempo, eu investiria em melhorias principalmente relacionadas à experiência do usuário e refinamento visual. Adicionaria transições e animações mais suaves para tornar a navegação mais fluida e agradável, além de melhorar estados de hover e feedback visual. Além disso, refinaria alguns espaçamentos e tipografia (em especial na versão para mobile) para deixar o layout mais próximo de um nível de produção.
